# Агент Бази Знань

## Огляд

Агент Бази Знань — це комплексна система управління знаннями, яка автоматично витягує, структурує та індексує цінну інформацію з повідомлень спільноти. Система надає можливості семантичного пошуку, публічну веб-енциклопедію та адміністративні інструменти для курування знань.

## Основні можливості

### 🤖 Автоматичне витягування знань
- Аналіз пакетів повідомлень з Telegram чатів
- Визначення цінного контенту за допомогою LLM
- Структурування знань у форматі Markdown
- Автоматична категоризація та тегування

### 🔍 Гібридний пошук
- Комбінація BM25 та семантичного пошуку
- Підтримка української мови з спеціальним аналізатором
- Ранжування результатів за релевантністю
- Швидкий пошук по великій базі знань

### 📚 Веб-енциклопедія
- Публічний доступ до бази знань за адресою `/wiki`
- Ієрархічна навігація по категоріях
- Інтерактивний пошук з миттєвими результатами
- Посилання на оригінальні повідомлення в Telegram

### ⚙️ Адміністративна панель
- Управління записами знань (створення, редагування, видалення)
- Налаштування інструкцій для агента
- Моніторинг черги помилок (DLQ)
- Режим попереднього перегляду для тестування

## Архітектура системи

```

```

## Процес обробки повідомлень

### 1. Формування пакетів (Chunking)
- **Часові вікна**: 15-хвилинні інтервали
- **Розмір пакету**: максимум 50 повідомлень
- **Перекриття**: 5 повідомлень між пакетами для контексту

### 2. Аналіз цінності
Агент визначає, чи містить пакет повідомлень корисні знання:
- ✅ Технічні рішення та інструкції
- ✅ Найкращі практики та рекомендації
- ✅ Корисні посилання та ресурси
- ✅ Рішення спільноти та обговорення
- ❌ Звичайні розмови та привітання
- ❌ Особиста інформація

### 3. Витягування структурованих даних
При виявленні цінного контенту агент створює:
- **Назва**: Стисла назва українською (5-10 слів)
- **Зміст**: Структурований Markdown з заголовками
- **Теги**: Ключові слова для пошуку
- **Категорія**: Technology, Business, Community, Events, Resources, Other
- **Шлях дерева**: Ієрархічна класифікація (наприклад, "Technology/PHP/Symfony")

### 4. Збагачення метаданими
- Посилання на оригінальні повідомлення в Telegram
- Інформація про автора та час створення
- Тип джерела (telegram, manual, etc.)

## Використання веб-енциклопедії

### Навігація по дереву знань
1. Відкрийте `/wiki` у браузері
2. Використовуйте ліву панель для навігації по категоріях
3. Клікніть на категорію, щоб побачити підкатегорії та записи
4. Натисніть на запис для перегляду повного змісту

### Пошук знань
1. Введіть запит у пошукове поле вгорі сторінки
2. Система знайде релевантні записи за змістом та тегами
3. Результати ранжуються за релевантністю
4. Клікніть на результат для перегляду повного запису

### Перехід до джерела
- Кожен запис містить посилання "Перейти до джерела"
- Посилання веде до оригінального повідомлення в Telegram
- Дозволяє побачити повний контекст обговорення

## Адміністрування

### Доступ до адмін-панелі
Адміністративна панель доступна за адресою `/admin/knowledge` та містить чотири основні розділи:

#### 1. Записи (Entries)
- Перегляд усіх записів знань у табличному вигляді
- Створення нових записів вручну
- Редагування існуючих записів
- Видалення застарілих або неточних записів
- Фільтрація по категоріях через дерево знань

#### 2. Налаштування (Settings)
- **Перемикач енциклопедії**: Увімкнення/вимкнення публічного доступу
- **Базові інструкції**: Редагування системного промпту для агента
- **Інструкції безпеки**: Перегляд незмінних правил безпеки (тільки читання)

#### 3. DLQ Монітор
- Перегляд кількості повідомлень у черзі помилок
- Повторна обробка проблемних повідомлень
- Моніторинг стану обробки

#### 4. Перевірка (Preview)
- Тестування витягування знань без збереження
- Введення JSON-масиву повідомлень
- Перегляд результату роботи агента

### Налаштування інструкцій агента

Базові інструкції визначають, як агент аналізує та витягує знання:

```
Ти є асистентом для вилучення знань з української технічної спільноти. 
Твоя роль — аналізувати пакети повідомлень з Telegram та витягувати 
цінні, структуровані знання.

Фокусуйся на:
- Технічних рішеннях та інструкціях
- Найкращих практиках розробки
- Корисних ресурсах та посиланнях
- Рішеннях спільноти та обговореннях

Ігноруй:
- Звичайні розмови та привітання
- Особисту інформацію
- Спам та нерелевантний контент
```

## API для розробників

### A2A Інтеграція
Агент підтримує A2A протокол платформи для інтеграції з іншими компонентами:

```json
{
  "agent": "knowledge-base",
  "request": {
    "intent": "search_knowledge",
    "payload": {
      "query": "Symfony CORS налаштування",
      "limit": 5
    }
  }
}
```

### REST API Endpoints
- `GET /api/v1/knowledge/search` - Пошук знань
- `GET /api/v1/knowledge/tree` - Дерево категорій
- `GET /api/v1/knowledge/entries/{id}` - Отримання запису
- `POST /api/v1/knowledge/upload` - Завантаження повідомлень для обробки

### OpenAPI Специфікація
Повна документація API доступна за адресою `/api/v1/knowledge/openapi.json`

## Моніторинг та обслуговування

### Health Check Endpoints
- `/health/worker` - Стан обробника повідомлень
- `/health/dlq` - Стан черги помилок

### Метрики продуктивності
- **Швидкість обробки**: Час від повідомлення до індексованого знання
- **Якість витягування**: Відсоток цінних знань з оброблених пакетів
- **Швидкість пошуку**: Час відповіді на пошукові запити
- **Доступність системи**: Uptime енциклопедії та API

### Налаштування продуктивності
```env
# Кількість одночасних обробників
KNOWLEDGE_WORKER_CONCURRENCY=2

# Обмеження швидкості LLM викликів
KNOWLEDGE_RATE_LIMIT_MAX_TOKENS=60
KNOWLEDGE_RATE_LIMIT_REFILL_RATE=60
```

## Безпека та конфіденційність

### Захист даних
- Автоматичне виключення особистих даних (телефони, email, адреси)
- Валідація посилань на повідомлення
- Доступ до адмін-функцій тільки для авторизованих користувачів

### Безпека LLM
- Незмінні інструкції безпеки для всіх запитів
- Фільтрація та валідація контенту
- Обмеження швидкості для запобігання зловживанням
- Аудит-логування всіх взаємодій з LLM

## Поширені проблеми та рішення

### Агент не витягує знання з повідомлень
1. Перевірте, чи містять повідомлення технічний контент
2. Переконайтеся, що базові інструкції налаштовані правильно
3. Використайте режим перевірки для тестування

### Пошук не знаходить релевантні результати
1. Спробуйте різні ключові слова
2. Перевірте, чи правильно налаштований український аналізатор
3. Переконайтеся, що знання правильно проіндексовані

### Енциклопедія недоступна
1. Перевірте налаштування в адмін-панелі
2. Переконайтеся, що перемикач енциклопедії увімкнений
3. Перевірте логи на наявність помилок

### Повідомлення накопичуються в DLQ
1. Перевірте підключення до LLM сервісу
2. Переконайтеся, що не перевищено ліміти швидкості
3. Використайте функцію повторної обробки в адмін-панелі
